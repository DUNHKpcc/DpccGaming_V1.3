# 内容发布规范

本规范用于统一 Blog 与 Docs 的内容组织方式，减少重复沟通。

## 1. 文档命名

1. 使用小写英文和连字符，例如 `deploy-checklist.md`
2. 文件名应与路由/数据配置保持一致
3. 避免中文文件名，降低跨平台编码问题

## 2. 封面规范

- 推荐比例：16:9
- 建议格式：`.webp`
- 建议分辨率：1280x720 或以上

> 如果封面缺失，页面会展示占位加载效果，建议发布前检查资源路径。

## 3. 发布步骤

1. 在 `public/docs` 新增对应 `.md` 文件
2. 在 `src/data/docsList.js` 注册卡片配置（标题、摘要、封面、路径）
3. 本地执行构建检查，确认无报错

## 4. 常见问题

- 文档打不开：检查 `file` 路径是否以 `/doc-files/` 开头
- 封面不显示：检查 `cover` 资源是否存在于 `public` 下
- 格式异常：确认 Markdown 标题、列表、代码块语法正确
