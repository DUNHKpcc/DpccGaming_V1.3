# 上线检查清单

每次发布前请按以下步骤完成检查，确保上线稳定性。

## 1. 配置检查

- 环境变量已同步到目标环境
- API 基础地址与部署环境一致
- 关键密钥未写入前端代码

## 2. 构建检查

```bash
npm run build
```

确认构建成功，并且无高风险警告。

## 3. 功能回归

1. 首页、游戏区、Blog、Docs 可正常访问
2. 登录和鉴权流程正常
3. 文档卡片点击后能正确加载 Markdown

## 4. 回滚准备

- 保留上一版可运行构建产物
- 准备数据库回滚脚本（如有结构变更）
- 明确负责人和应急沟通渠道

## 5. 发布后观测

- 关注错误日志与接口响应时间
- 关注白屏、资源 404 与权限报错
- 在 30 分钟内完成首次稳定性复核
